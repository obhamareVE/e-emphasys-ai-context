42I08 - Combined expression in LIKE predicate

An SQL statement contains a LIKE predicate which left hand side is an explicit row value constructor (also called a Combined). The left hand side of a LIKE predicate must be a single expression that is not contained in an explicit row value constructor.

Example
select *
from   dbtst120
where  { workdept } LIKE "A.*"

This query will cause the error:

SQLState 42I08: Combined expression in LIKE predicate

It can be solved by rewriting the expression as follows:

select *
from   dbtst120
where  workdept LIKE "A.*"
