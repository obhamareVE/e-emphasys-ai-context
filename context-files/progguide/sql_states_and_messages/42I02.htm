42I02 - Ambiguous column reference

An SQL statement contains a column reference that does not uniquely identify a column.

Example
select *
from   dbtst120, dbtst180
where  empno refers to dbtst180

Table dbtst120 and dbtst180 both contain a column with the name empno. This query will cause the error:

SQLState 42I02: Ambiguous column reference 'empno'

It can be solved by prefixing the column name with the table name or alias name: 

select *
from   dbtst120, dbtst180
where  dbtst120.empno refers to dbtst180
