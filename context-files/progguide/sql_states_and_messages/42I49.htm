42I49 - Array values can only be compared for equality

An SQL statement contains an array column (without indexing operator) in a BETWEEN predicate, in an INRANGE predicate or in a comparison predicate with a comparison operator other than '='.

Example 1
select a.firstnme, a.lastname, a.phoneno
from   dbtst120 a, dbtst120 b
where  a.phoneno > b.phoneno
and    b.empno = 10

You can apply any comparison operator to an individual array element, but equality ('=') is the only comparison operator that you can apply to an array column as a whole. Because column phoneno is an array column, this query will cause the error:

SQLState 42I49: Array values can only be compared for equality

Example 2
select firstnme, lastname, phoneno
from   dbtst120
where  phoneno between '010' and '050'

The between expression in this query is equivalent to the expression: 

phoneno >= '010' and phoneno <= '050'

Because equality is the only comparison operator that can be applied to an array column as a whole, this query will cause the error:

SQLState 42I49: Array values can only be compared for equality
