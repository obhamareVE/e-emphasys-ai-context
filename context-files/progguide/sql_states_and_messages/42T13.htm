42T13 - Type conflict on parameter

An SQL statement contains a parameter and two conflicting types have been derived for this parameter.

Example
select firstnme
from   dbtst120
where  firstnme = :var
and    salary = :var

Column firstnme is of type string and column salary is of type real. According to the condition 'firstnme = :var' parameter var must be of type string, but according to the condition 'salary = :var' parameter var must be of type real. This will cause the error:

SQLState 42T13: Type conflict on parameter. Parameter type is STRING, type REAL expected
