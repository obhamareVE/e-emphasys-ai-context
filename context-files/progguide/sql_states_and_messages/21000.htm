21000 - Cardinality violation

An SQL statement contains a comparison predicate of which a value expression is a scalar subquery, and the subquery results in more than 1 row.

Example
select * from dbtst120 where empno = ( select empno from dbtst120 )

The subquery should result in at most 1 row, but it results in more than 1 row. It will cause the error:

SqlState=21000, '', NativeError=310

It can be solved by assuring that the subquery results in at most 1 row. 

select * from dbtst120 where empno = ( select empno from dbtst120 where emnpo = 10 )
