42I82 - Syntax conflicts with Parse mode

A SQL statement contains a predicate that is only supported in another parse mode.

Examples
select firstnme
from   dbtst120 
where  firstnme like 'C/_%' escape '/'

Additional information: ANSI syntax for LIKE predicate requires parse mode 'ansi'.

The ESCAPE keyword can only be used for queries that are parsed in ANSI mode. 

select firstnme
from   dbtst120 
where  firstnme alike 'C%'

Additional information: ALIKE predicate is not allowed in parse mode 'ansi'.

The ALIKE keyword can only be used for queries that are parsed in non-ANSI mode. 

SELECT workdepts.deptname, employees.lastname
FROM dbtst100 workdepts, dbtst120 employees
WHERE workdepts.deptno REFERS TO employees.workdept

Additional information: REFERS TO predicate is not allowed in parse mode 'ansi'.

The REFERS TO predicate can only be used for queries that are parsed in non-ANSI mode.
