42I13 - Columns in combined reference operand cannot be of different tables

An SQL statement contains a REFERS TO predicate, the left hand side of this predicate is an explicit row value constructor (als called a Combined) and the elements of this row value constructor contain columns of different tables.

Example
select *
from   dbtst160, dbtst100, dbtst180
where  dbtst160.deptno refers to dbtst100
and    { dbtst160.projno, dbtst100.mgrno } refers to dbtst180

If the left hand side of a REFERS TO predicate contains only columns then these columns must all be of the same table. In the second REFERS TO predicate of the above query the left operand contains a column of table dbtst160 and one of dbtst100. This query will cause the error:

SQLState 42I13: Columns in combined reference operand cannot be of different tables
