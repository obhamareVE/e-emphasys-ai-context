42I50 - Invalid use of array value in expression

An SQL statement contains an array column (without indexing operator):

  • in an expression with a binary operator;
  • as the start or length value in a substring expression, or
  • as the index value in an array indexing expression
Example 1
select firstnme, lastname, phoneno
from   dbtst120
where  "0342" & phoneno = "0342-12345"

Column phoneno is an array column. An array column as a whole cannot be used in an expression with a binary operator (concatenation in this case). Only individual array elements can be used in such expressions. This query will cause the error:

SQLState 42I50: Invalid use of array value in expression

Example 2
select a.empno, a.firstnme
from   dbtst120 a, dbtst190 b
where  a.empno = b.empno
and    a.firstnme(1;b.longarr) = "JOHN"

The column longarr is an array column. An array column cannot be used as the start or length value of a substring operator. This query will cause the error:

SQLState 42I50: Array value not allowed in substring expression

Example 3
select a.empno, a.firstnme
from   dbtst120 a, dbtst190 b
where  a.empno = b.empno
and    a.phoneno(b.longarr) = "15972"

The columns phoneno and longarr are both array columns. An array column cannot be used as the index value of an indexing operator. This query will cause the error:

SQLState 42I50: Index in array indexing operator is an array value
