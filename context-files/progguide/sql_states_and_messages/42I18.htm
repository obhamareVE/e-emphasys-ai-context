42I18 - Table is referred to by different tables

An SQL statement contains two or more REFERS TO predicates that refer to the same table, but with columns of different tables.

Example
select *
from   dbtst120, dbtst100, dbtst160
where  dbtst100.mgrno refers to dbtst120
and    dbtst160.respemp refers to dbtst120

Both REFERS TO predicates refer to the same table dbtst120, but the first is done with a column of dbtst100 and the second with a column from dbtst160. This query causes the error:

SQLState 42I18: Table 'dbtst120' is referred to by different tables

It is possible to join different tables to the same table within one query, but each table should then join with a different instance of that table: 

select *
from   dbtst120 tst120a, dbtst120 tst120b, dbtst100, dbtst160
where  dbtst100.mgrno refers to tst120a
and    dbtst160.respemp refers to tst120b
