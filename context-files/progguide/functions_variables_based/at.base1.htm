at.base()
Syntax:

function long at.base  (string basic_variable[,arg...], string based_variable[,arg...])

Description

Use this function to base one variable (the based variable) on another variable (the basic variable). The two variables then use the same memory area.

Arguments
string basic_variable[,arg...]

The variable on which the based_variable is to be based. You can pass arguments to this variable. These specify the start position of the based variable within the basic variable. The default start position is 1.

string based_variable[,arg...]

The based variable. You can pass arguments to this variable. These specify the length of the based variable. The length plus the start position must not exceed the length reserved for the basic variable.

If you pass the argument -1 to the variable, the based variable uses the memory space of the basic variable from the specified start position to the end position. If this argument is 0, the variable uses the memory space of the basic variable from the specified start position for the declared length of the based variable.

This variable must be declared as BASED.

 

Context

This function is implemented in the porting set and can be used in all script types.

See also

at.base()

Notes

If the variables are multidimensional long or double arrays, only the first element can differ in the declarations. This is because the array values are stored column by column.

Do not reallocate basic_variable (with alloc.mem()). Doing this can cause serious problems, such as a core dump or corrupted memory.

Example

In this example, the contents of the variable long_to_be_based(1, 1) are equal to the value of basic_long(4, 1). That is, a matrix of 2 by 5 longs. 

long basic_long(5, 5)
long long_to_be_based(3, 5) based

at.base(basic_long, 4, 1, long_to_be_based, -1, 0)
