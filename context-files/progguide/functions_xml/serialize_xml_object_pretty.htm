Serialize XML Object Formatted
Syntax:

function long xmlWritePretty  (long fp, long fromNode [, long toNode])

Description

Serialize an XML object to XML and write resulting XML document to an open stream. Any string information in the XML document is converted to UTF-8 format.

The XML data is formatted by adding new-lines and tabs to make it human readable. Use Serialize XML Object (xmlWrite) instead if the XML data does not need formatting.

Arguments
long fp

fp must be a file pointer opened for write obtained from a call to seq.open(), pipe.open(), ims.openfba() or ims.openvba().

long fromNode

See Handling of fromNode and toNode arguments
[long toNode]

See Handling of fromNode and toNode arguments

 

Return values
>= 0 Success; Number of bytes written to output when successful
< 0

Error. As many bytes as possible are written to fp. The place where the output of bytes stops, corresponds to the place in the XML tree where the first serialization error occurred. Some specific values:

-1 Incorrect argument. For example, file pointer fp is closed, or fromNode or toNode is not a valid xml node.
-3 Cannot write to fp (e.g. it is open for reading instead of writing), or a TSS conversion error occured. If TSS conversion of a certain text element (e.g. an attribute name or value) fails, then the output of bytes stops before that text element.
-4 An invalid character was encountered. The output of bytes stops just before that character. See the xmlContainsValidCharactersOnly() function.

 

 

Context

This function is implemented in the 4GL Tools and can be used in all script types.
