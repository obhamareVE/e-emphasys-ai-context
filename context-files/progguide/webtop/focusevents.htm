Focus events

The WebTop will detect if a session needs focus events (by checking if the following sections are used: check.input, before.field and before.input). If a session does not need focus events, then the WebTop will not send focus events for this session. This results in better performance when for example tabbing through the fields of such a session. This however has the following consequence:

Normally, when a field receives the focus, the field is automatically redisplayed. This will not happen when no focus events are send anymore. This means that if you change values of table fields, you must always call the display function. You can not rely on the automatic redisplay of fields when the field receives the focus. Note: this only applies to sessions for which the WebTop decides not to send focus events!
