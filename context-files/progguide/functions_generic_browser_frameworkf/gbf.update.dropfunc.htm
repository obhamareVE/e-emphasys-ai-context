gbf.update.dropfunc()
Syntax:

#include <bic_gbf>

function long gbf.update.dropfunc  (long obj.id, long drop.function.id)

Description

This function is actual a define and is defined as:

gbf.update.dropfunc(obj.id, drop.func) gbf.update.object(obj.id, GBF.NO.UPD.DESC, GBF.NO.UPD.ICON, GBF.NO.UPD.FUNC, drop.func, GBF.NO.UPD.MENU)

 

See gbf.update.object() * for more details.

Arguments
long obj.id

The obect identification
long drop.function.id

If GBF.USE.DEF.FUNC then use the default function, otherwise

If GBF.NO.UPD.FUNC, then this function will not be changed,

else this function will be used as the new drag and drop function (see gbf.add.object() for what the drag and drop function is supposed to do).

 

Return values
= 0 Successful completion
GBF.ILL.ICON Illegal icon.set identification
GBF.ILL.FUNCTION Illegal default.function.id or drop.function.id value
GBF.ILL.OBJECT Illegal obj.id given
GBF.ILL.STATE GBF is not in the right state to deal with this function

 

Context

This function is implemented in the 4GL Engine and can be used in all script types.

gbf.update.dropfunc(), gbf.update.desc(),gbf.update.icon(), gbf.update.menumask(), gbf.update.object()
Restriction

This functions may only be called by the application when the GBF issues a call back to the main application, thus on a gbf.drag.drop(), gbf.menu.selected(), gbf.help.selected() or gbf.menu.selected() call. The object to be updated is identified by the given obj.id.
