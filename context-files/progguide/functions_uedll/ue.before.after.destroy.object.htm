ue.before.after.destroy.object()
Syntax:

function extern long ue.before.after.destroy.object  ()

Description

This hook is called in case data is deleted, either by the user pressing the delete button on the UI, or programmatically when one of the functions dal.destroy.object() or dal.destroy() is called.

Note that this hook is also executed in case a 'stand-alone' db.delete() is done.

In case a DAL is present, this hook is executed just before the after.destroy.object() hook.

In case no DAL is present, this hook is executed just before the main.table.io:after.delete: UI script section.

In case of a 'stand-alone' db.delete(), this hook is executed just after the db.delete() is done.

Return values

This hooks returns 0 if deleting the record is permitted. It should return a negative value like DALHOOKERROR, in case of an error.

Context

This function is implemented in the 4GL Engine and can be used in UEDLL script types.

Example
function extern long ue.before.after.destroy.object()
{
        | Do some extra actions just before the standard 
        | after.destroy.object hook
        ...
        return(0)
}
