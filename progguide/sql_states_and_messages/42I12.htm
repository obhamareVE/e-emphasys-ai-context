42I12 - Combined reference operand must contain only constants or only columns

An SQL statement contains a REFERS TO predicate, the left hand side of this predicate is an explicit row value constructor (als called a Combined) and the elements of this row value constructor contain both constants and columns.

Example
select *
from   dbtst160 a, dbtst180 b
where  { a.respemp, a.projno, date.to.num(2002,1,1), date.to.num(2003,1,1) } refers to b

The left hand side of a REFERS TO predicate must contain only constants or only columns, but it cannot contain a mixture of constants and columns. The left hand side of the REFERS TO predicate in above query contains two columns and two constants. This query will cause the error:

SQLState 42I12: Combined reference operand must contain only constants or only columns
