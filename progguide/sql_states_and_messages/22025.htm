22025 - Invalid escape sequence

An SQL statement contains an ansi LIKE predicate with an invalid ESCAPE sequence in the pattern.

Example
select * from dbtst120 where firstnme alike 'c_%!' escape '!'

The pattern contains an escape character that is not followed by '_', '%' or the escape character itself.

SqlState=22025, '', NativeError=325

It can be solved by assuring that the escape character in the like pattern is always followed by '_', '%' or the escape character. 

select * from dbtst120 where firstnme alike 'c!_%' escape '!'
