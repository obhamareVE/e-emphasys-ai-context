42I65 - Table has child tables

The table in the FROM clause of an SQL DELETE statement has a child table; the reference mode of this child table is mandatory or mandatory unless empty and the reference is maintaind through lookup, through reference counters or by means of a cascade action on the child table.

Example
delete from dbtst120
where  empno = 10

There is a reference from dbtst160.respemp to dbtst120.empno. The reference mode is mandatory unless empty and is maintained through counters. This means that this delete would require reference checking. An SQL DELETE statement can only be executed if no reference checking needs to be done. This statement will cause the error:

SQLState 42I65: Table 'dbtst120' has child tables
