22008 - Datetime value out of range

An SQL statement contains a comparison of a date column with an integer contant or a real constant and the value of the constant is larger than 3652059.

Example
select *
from dbtst120
where hiredate = 1234567890;

The integer constant has value 1234567890, which is bigger than 3652059. This will result in the following error:

SqlState=22008, '', NativeError=205

It can be solved by assuring that the integer constant is smaller than or equal to 3652059. 

select * from dbtst120 where hiredate = 710886

Note: The behavior on the example is different for different implementations of the SQL language.
