42T21 - Incompatible types for UNION column

An SQL statement contains a UNION operator and in the two queries that are combined by this UNION operator there are two corresponding select items that do not have comparable datatypes.

Example
select firstnme
from   dbtst120
union  all
select salary
from   dbtst120

In this UNION the columns firstnme and salary are corresponding select items. These columns do not have comparable datatypes because column firstnme is of type string and column salary is of type real. This will cause the error:

SQLState 42T21: Incompatible types for UNION column 1
