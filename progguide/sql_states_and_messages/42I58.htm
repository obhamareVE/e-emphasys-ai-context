42I58 - Statement exceeds maximum number of subqueries

An SQL statement contains a subquery which nesting level is larger than the maximum allowed nesting level of subqueries.

Example
select empno from dbtst120 t0 where exists (
 select empno from dbtst120 t1 where exists (
  select empno from dbtst120 t2 where exists (
   select empno from dbtst120 t3 where exists (
    select empno from dbtst120 t4 where exists (
     select empno from dbtst120 t5 where exists (
      select empno from dbtst120 t6 where exists (
       select empno from dbtst120 t7 where exists (
        select empno from dbtst120 t8 where exists (
         select empno from dbtst120 t9 where exists (
          select empno from dbtst120 t10 empno = 10 ))))))))))

The nesting level of the main query is 0. The nesting level of a subquery is the nesting level of its parent query plus 1. The maximum allowed nesting level is 9. In this query, the most nested query (the subquery on table t10) has nesting level 10. This will cause the error:

SQLState 42I58: Statement exceeds maximum number of subqueries: 9

There is no limit on the number of subqueries there is only a limit on the nesting level. The following query is a correct query: 

select empno from dbtst120 t0 where
 exists ( select empno from dbtst120 t1 ) and
 exists ( select empno from dbtst120 t2 ) and
 exists ( select empno from dbtst120 t3 ) and
 exists ( select empno from dbtst120 t4 ) and
 exists ( select empno from dbtst120 t5 ) and
 exists ( select empno from dbtst120 t6 ) and
 exists ( select empno from dbtst120 t7 ) and
 exists ( select empno from dbtst120 t8 ) and
 exists ( select empno from dbtst120 t9 ) and
 exists ( select empno from dbtst120 t10 )
