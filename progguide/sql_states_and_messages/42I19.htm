42I19 - Column is not a grouping column

The SELECT clause, the WHERE clause or the HAVING clause of a SQL statement uses a column of a grouped query, but the column is not used within an aggregate function and the column it also not a grouping column (the column is not in the GROUP BY clause)

Example 1
select edlevel, salary, max(bonus)
from   dbtst120
where  workdept = "A01"
group  by edlevel

Column salary is not in the GROUP BY clause, therefore it cannot be used in the SELECT clause, unless (like bonus) it is used in an aggregate function. The WHERE clause filters the records before grouping is applied, so column workdept can be used here without an aggregate function. This query will cause the error:

SQLState 42I19: Column 'salary' is not a grouping column

Example 2
select *
from   dbtst120
group  by workdept

The select item * or <table name>.* cannot be used if the query contains a GROUP BY clause, because these items will automatically select the column _compnr as well and this column cannot appear in a GROUP BY clause. This query will cause the error:

SQLState 42I19: Column 'dbtst120._compnr' is not a grouping column
