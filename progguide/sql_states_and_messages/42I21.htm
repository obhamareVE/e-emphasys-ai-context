42I21 - Aggregate function applied to column which is not a column of a grouped table

An SQL statement contains an aggregate function that is applied to a column which is not a column of a grouped table.

Example 1
select max(edlevel)
from   dbtst120
where  avg(edlevel) > 10
group  by edlevel

The query does contain a GROUP BY clause, but a WHERE clause always filters the records before grouping is applied. In the context of the WHERE clause table dbtst120 is not a grouped table and the aggregate function AVG cannot be applied to column edlevel. This query will cause the error:

SQLState 42I21: Aggregate function applied to column 'edlevel', which is not a column of a grouped table

Example 2
select workdept, max(salary)
from   dbtst120
where  count(*) > 10
group  by workdept

The aggregate function COUNT(*) cannot be used in the WHERE clause, because it is not applied to a grouped result. The HAVING clause must be used to filter records of the grouped result. This query will cause the error:

SQLState 42I21: COUNT(*) cannot be applied to a non grouped query
