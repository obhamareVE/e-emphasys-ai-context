Macros - transaction header
Warning

The functions and macros listed below must not be used anymore. Using them in BaanERP 5.0c or higher releases may result in less reliable data. Know that there is a new set of functions that you can use for retrieval of audit information. Refer to Audit management overview

Use these macros for mapping transaction header information. All the macros take as their input. This is filled by calling aud_read_tran() or aud_read_next_tran().

aud_get_tran_bytes() 
long aud_get_tran_bytes( string tran_hdr() )

This returns the total size in bytes of the transaction data.

aud_get_tran_commit_date() 
string aud_get_tran_commit_date( string tran_hdr() )

This returns the UTC commit date from the transaction header.

aud_get_tran_commit_time() 
string aud_get_tran_commit_time( string tran_hdr() )

This returns the UTC commit time from the transaction header.

aud_get_tran_entries() 
long aud_get_tran_entries( string tran_hdr() )

This returns the number of audited actions in the transaction.

aud_get_tran_id() 
long aud_get_tran_id( string tran_hdr(), long g1, long g2)

This reads the transaction id from the transaction header. The transaction id consists of two longs which together form a unique identification.

aud_get_tran_sess() 
string aud_get_tran_sess( string tran_hdr() )

This returns the name of the session that committed the transaction.

aud_get_tran_status() 
long aud_get_tran_status( string tran_hdr() )

This returns the commit status of the transaction. Possible values are: 

AUD_COMMITTED   transaction committed
AUD_ABORTED     transaction canceled
AUD_PREPARED    it is not certain whether the transaction has
                   been committed or aborted
aud_get_tran_tuname() 
string aud_get_tran_tuname( string tran_hdr() )

This returns the BAAN user name of the user who performed the transaction.

aud_get_tran_uuid() 
long aud_get_tran_uuid( string tran_hdr() )

This returns the UNIX user id of the user who performed the transaction.
