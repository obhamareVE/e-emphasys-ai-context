set.refresh.rate()
Syntax:

function long set.refresh.rate  (long rateInSeconds [, long thresholdInSeconds])

Description

With this function the refreshing rate of the session will be overruled temporary.

The refresh interval is part of the User Data Template.

The interval is standard used for sessions where the Auto Refresh in standard commands is set.

This function is overruling the setting of the session, also for users without refresh interval.

The windows of same connection/browser will be refreshed in the minimum refreshrate seconds of all users active sessions.

Each session of the back-end is using a timer(g.timer) to create a timer event for the choice.interupt.

At change of the refreshing rate, the front-end of that session is informed by tag SESSION REFRESH in the RESPONSE note.

This function will create an assert error when in a session without AutoRefresh to ensure the presence of the choice subsections choice.interrupt.

Other values are returned in the error value.

This function is available from TIV level 1800.

This function is not supported under WorkTop and WebUI version before AP0068 resp. BL0004 (Aug 2012). (see Return Value)

Arguments
long rateInSeconds

The refreshing rate of the session.

Special values:

Zero: the session will set to it's the original refreshing rate.

-1: the refreshing of the session will be stopped.
[long thresholdInSeconds]

The duration of the refreshing rate:

0 - the refreshRate has no end time

> 0 - the refreshRate will be changed for thresholdInSeconds.

After thresholdInSeconds, the original refreshRate is restored.

 

Return values

Error Code:

0 - Successful completion

-1 - Refresh rate must be greater equal -1.

-2 - Threshold must be greater equal zero.

-3 - No action on XML Response Node "Session Refresh" ( thin-client level before TIV level 1030 )

-4 - refresh interval not supported without ThinClient

Context

This function is implemented in the 4GL Engine and can be used in 4GL script types.
