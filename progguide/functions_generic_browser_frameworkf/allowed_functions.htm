Allowed functions

The Generic Browser Framework (GBF) goes through several states during its lifecycle from the start till the end of a session.

State Description
GBF chaos (Yellow) This state indicates that the main session has started, but no actions to the GBF have been done yet, hence the state of the GBF is undetermined and no functions, except gbf.init() should be used.
GBF initialized (Blue) This state indicates that a gbf.init() has been received, which means that the GBF is initializing itself, and waiting for a gbf.start(). In this state the main session is still in control.
GBF running (Green) This state indicates that now the GBF is in control, due to a gbf.start(). The GBF will be in control until it is told to terminate.
GBF terminated (Red) This state indicates that the GBF just has relinquished control and handed it back to the main session. In this state some settings can still be retrieved from the GBF. The GBF waits for session termination or for a new gbf.init().

In the GBF running state there are further restrictions if the yes is followed by a sequence of numbers. These numbers refer to:

1. gbf.bms.received()
2. gbf.drag.drop()
3. gbf.get.children()
4. gbf.get.top.level()
5. gbf.menu.selected()
6. gbf.help.selected()
7. gbf.save()

 

It can be that this is not enforced by the GBF, so for example a gbf.get.first.child() call during a gbf.get.top.level()can be run without an error.
