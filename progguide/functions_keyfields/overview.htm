Key fields Object overview

For data synchronization between sessions and for drag/drop operations between sessions a common format is needed to exchange information about database records. For this purpose the key fields object can be used. This object can hold the name/value pairs of the primary key fields of a database record. As a data storage mechanism the XML object is used. Because drag/drop operations require the exchange of sets of these objects, also a key fields container is used.

These functions can be used in Composite Child sessions but can also be used in other session types.
