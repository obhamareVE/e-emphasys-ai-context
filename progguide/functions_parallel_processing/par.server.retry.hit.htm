par.server.retry.hit()
Syntax:

function void par.server.retry.hit  ()

Description

This function must be called in a server after a real db.retry.hit() call. When this function is called x times and x = MAX.RETRY-1, this function will do an abort of the client server communication with the abort message "Too many retry hits in server". See section: Parallel Application Processing Database Retries for a further explanation.

Context

This function is implemented in the 4GL Engine and can be used in 3GL script types.
